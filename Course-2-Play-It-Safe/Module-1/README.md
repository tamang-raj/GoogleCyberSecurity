# 📚 Topics Covered

## CISSP Security Domains

This module explored the **eight CISSP security domains**, which form a globally recognized framework for information security.

### The Eight CISSP Domains

| Domain | Description |
|---------|-------------|
| Security and Risk Management | Governance, policies, compliance, and risk management |
| Asset Security | Protecting physical and digital assets |
| Security Architecture and Engineering | Secure system and infrastructure design |
| Communication and Network Security | Securing network infrastructure and communications |
| Identity and Access Management (IAM) | Authentication, authorization, and access control |
| Security Assessment and Testing | Security audits, testing, and vulnerability assessments |
| Security Operations | Incident response, monitoring, and disaster recovery |
| Software Development Security | Integrating security into the software development lifecycle |

## Threats, Risks, and Vulnerabilities

### Threat

Anything capable of exploiting a weakness and causing harm to an organization.

Examples:

- Malware
- Insider threats
- Phishing attacks
- Ransomware
- Social engineering

---

### Vulnerability

A weakness in a system, application, process, or configuration that can be exploited by a threat.

Examples:

- Weak passwords
- Outdated software
- Misconfigured systems
- Unpatched vulnerabilities

---

### Risk

The potential for loss or damage when a threat exploits a vulnerability.

Examples:

- Data breaches
- Financial loss
- Operational disruption
- Reputational damage

---

## NIST Risk Management Framework (RMF)

This module introduced the **NIST RMF**, a structured approach to managing cybersecurity risks.

The framework includes:

1. Prepare
2. Categorize
3. Select
4. Implement
5. Assess
6. Authorize
7. Monitor
