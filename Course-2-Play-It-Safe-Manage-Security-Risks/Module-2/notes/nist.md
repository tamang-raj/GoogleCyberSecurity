# NIST Cybersecurity Framework (CSF) & NIST SP 800-53

## Overview

The **National Institute of Standards and Technology (NIST)** develops cybersecurity frameworks and standards that help organizations identify, manage, and reduce cybersecurity risks.

These frameworks provide a structured approach to protecting systems, data, and critical assets from cyber threats.

---

# Why Organizations Use Security Frameworks

Security frameworks help organizations:

- Identify cybersecurity risks
- Protect sensitive data and assets
- Reduce threats and vulnerabilities
- Develop effective security policies
- Improve incident response
- Maintain compliance with industry standards

Frameworks serve as a **starting point** for building an organization's cybersecurity strategy.

---

# NIST Cybersecurity Framework (CSF)

## What is NIST CSF?

The **NIST Cybersecurity Framework (CSF)** is a **voluntary** framework that provides:

- Standards
- Guidelines
- Best practices

to help organizations manage and reduce cybersecurity risks.

It is widely used by businesses, government agencies, and organizations around the world.

---

# The Five Core Functions of NIST CSF

```text
          NIST CSF

      Identify
          │
          ▼
      Protect
          │
          ▼
       Detect
          │
          ▼
      Respond
          │
          ▼
      Recover
```

---

# 1. Identify

## Purpose

Understand and manage cybersecurity risks affecting:

- People
- Systems
- Data
- Assets
- Business operations

## Activities

- Inventory hardware and software
- Identify critical assets
- Assess cybersecurity risks
- Understand business environment
- Monitor systems for security issues

### Example

A security analyst identifies outdated software running on company servers.

---

# 2. Protect

## Purpose

Implement safeguards to reduce cybersecurity risks.

## Activities

- Access control
- Security awareness training
- Encryption
- Multi-Factor Authentication (MFA)
- Data protection
- Security policies
- Firewall implementation

### Example

An organization enables MFA for all employee accounts to prevent unauthorized access.

---

# 3. Detect

## Purpose

Identify cybersecurity incidents as quickly as possible.

## Activities

- Continuous monitoring
- Log analysis
- SIEM monitoring
- Intrusion Detection Systems (IDS)
- Threat detection

### Example

A SIEM system detects repeated failed login attempts and alerts the security team.

---

# 4. Respond

## Purpose

Take action to contain and manage security incidents.

## Activities

- Incident response
- Communication
- Threat containment
- Root cause analysis
- Updating security procedures

### Example

The security team isolates an infected workstation to stop malware from spreading.

---

# 5. Recover

## Purpose

Restore systems and business operations after a cybersecurity incident.

## Activities

- Restore backups
- Recover data
- Repair affected systems
- Improve disaster recovery plans
- Resume normal operations

### Example

After a ransomware attack, the IT team restores encrypted files from secure backups.

---

# NIST SP 800-53

## What is NIST SP 800-53?

**NIST Special Publication (SP) 800-53** provides a comprehensive catalog of security and privacy controls.

It expands on the NIST CSF by offering detailed controls for protecting information systems, particularly within the **U.S. federal government**.

---

## Purpose of NIST SP 800-53

- Protect federal information systems
- Improve cybersecurity and privacy
- Manage security risks
- Support compliance requirements
- Strengthen organizational security

---

# Relationship with the CIA Triad

The security controls in **NIST SP 800-53** are designed to protect the three principles of the CIA Triad.

| CIA Principle | Goal |
|--------------|------|
| 🔒 Confidentiality | Prevent unauthorized access to information |
| ✅ Integrity | Ensure data remains accurate and trustworthy |
| ⚡ Availability | Ensure authorized users can access systems when needed |

---

# NIST CSF vs NIST SP 800-53

| NIST CSF | NIST SP 800-53 |
|-----------|----------------|
| Voluntary framework | Detailed security controls |
| Focuses on managing cybersecurity risks | Focuses on implementing security and privacy controls |
| Five core functions | Comprehensive catalog of safeguards |
| Used by organizations worldwide | Primarily designed for U.S. federal systems |

# Key Takeaways

- NIST develops widely adopted cybersecurity frameworks and standards.
- The **NIST Cybersecurity Framework (CSF)** is a voluntary framework based on five core functions:
  - Identify
  - Protect
  - Detect
  - Respond
  - Recover
- **NIST SP 800-53** provides detailed security and privacy controls, especially for U.S. federal information systems.
- Both frameworks help organizations improve their security posture and reduce cybersecurity risks.
- NIST SP 800-53 supports the **Confidentiality, Integrity, and Availability (CIA) Triad**.

# Real-World Example

A healthcare organization adopts the **NIST Cybersecurity Framework**:

- **Identify:** Inventory patient databases and medical devices.
- **Protect:** Encrypt patient records and require MFA for staff.
- **Detect:** Monitor network activity using SIEM and IDS.
- **Respond:** Isolate infected systems during a malware attack.
- **Recover:** Restore patient data from backups and resume healthcare services.

To strengthen these practices, the organization implements **NIST SP 800-53** security controls, ensuring the confidentiality, integrity, and availability of sensitive patient information.
