# Security Frameworks and Controls

## Overview

Security frameworks and security controls work together to protect an organization's data, systems, and people. Frameworks provide the overall strategy and guidelines, while controls are the specific safeguards used to reduce risks.

---

# Security Frameworks

## What is a Security Framework?

A **security framework** is a set of guidelines and best practices that helps organizations:

- Manage cybersecurity risks
- Protect sensitive data
- Maintain privacy
- Meet legal and regulatory compliance requirements
- Improve overall security posture

Frameworks provide a structured approach to planning, implementing, and maintaining security.

---

## Purpose of Security Frameworks

Security frameworks help organizations:

- Identify risks and threats
- Develop security policies
- Improve incident response
- Maintain compliance with regulations
- Protect confidentiality, integrity, and availability (CIA Triad)

---

## Common Security Frameworks

### 1. NIST Risk Management Framework (RMF)

A seven-step framework used to manage security and privacy risks.

Steps include:

1. Prepare
2. Categorize
3. Select
4. Implement
5. Assess
6. Authorize
7. Monitor

---

### 2. NIST Cybersecurity Framework (CSF)

Provides guidance for:

- Identifying cyber risks
- Protecting systems
- Detecting threats
- Responding to incidents
- Recovering from attacks

---

### 3. Cyber Threat Framework (CTF)

Provides a common language for:

- Describing cyber threats
- Sharing threat intelligence
- Improving communication among security professionals

---

### 4. ISO/IEC 27001

An internationally recognized framework for:

- Information Security Management Systems (ISMS)
- Protecting confidential information
- Managing security risks
- Continuous security improvement

---

# Security Controls

## What are Security Controls?

Security controls are safeguards that reduce specific security risks.

They help organizations:

- Prevent attacks
- Detect threats
- Reduce vulnerabilities
- Protect assets
- Support security frameworks

---

# Types of Security Controls

## 1. Physical Controls

Protect physical assets and facilities.

### Examples

- Security guards
- Gates
- Locks
- Security cameras
- Badge access systems
- Fences

---

## 2. Technical Controls

Technology-based safeguards that protect systems and data.

### Examples

- Firewalls
- Antivirus software
- Encryption
- Multi-Factor Authentication (MFA)
- Intrusion Detection Systems (IDS)
- Access control systems

---

## 3. Administrative Controls

Policies and procedures that guide employee behavior.

### Examples

- Security policies
- Employee security training
- Separation of duties
- Asset classification
- Incident response plans
- Password policies

---

# Frameworks vs Controls

| Security Framework | Security Control |
|--------------------|------------------|
| Provides overall security guidance | Implements specific protections |
| Strategic | Operational |
| Defines policies and processes | Enforces policies through safeguards |
| Helps organizations manage risks | Helps reduce individual risks |

```text
Security Framework
        │
        ▼

        │
        ▼

        │
        ▼
Implement Controls
        │
        ▼
Reduce Risks and Protect Assets
```

---

# Relationship with the CIA Triad

Both frameworks and controls help protect the three pillars of cybersecurity:

### Confidentiality

Ensure only authorized users access information.

Examples:

- Encryption
- MFA
- Access controls

---

### Integrity

Ensure data remains accurate and unchanged.

Examples:

- Hashing
- Digital signatures
- File integrity monitoring

---

### Availability

Ensure systems and data remain accessible.

Examples:

- Backups
- Disaster recovery
- Redundant servers
- UPS power systems

---

# Benefits of Security Frameworks and Controls

Organizations can:

- Improve security posture
- Reduce cyber risks
- Protect sensitive information
- Meet legal and regulatory requirements
- Strengthen incident response
- Build customer trust

---

# Key Takeaways

- Security frameworks provide structured guidance for managing cybersecurity.
- Security controls are the safeguards used to implement framework recommendations.
- Controls can be physical, technical, or administrative.
- Frameworks and controls work together to reduce organizational risk.
- Popular frameworks include NIST RMF, NIST CSF, Cyber Threat Framework (CTF), and ISO/IEC 27001.
- Implementing frameworks and controls strengthens an organization's overall security posture.

# Real-World Example

A bank adopts the **NIST Cybersecurity Framework (CSF)** to establish its cybersecurity strategy.

To implement the framework, it deploys the following controls:

- **Physical:** Security guards, surveillance cameras, secure server rooms
- **Technical:** Firewalls, encryption, MFA, IDS
- **Administrative:** Employee cybersecurity training, password policies, incident response playbooks

Together, the framework provides the roadmap, while the controls enforce security measures that protect the bank's systems, customer data, and operations.
