# Security Frameworks and Controls

## Overview

This module introduces **security frameworks** and **security controls**, explaining how organizations use them to reduce risks, protect assets, and improve their overall security posture.

---

# Security Frameworks

## What are Security Frameworks?

Security frameworks are structured guidelines that help organizations:

- Build strong security policies
- Develop security processes
- Reduce security risks
- Protect data privacy
- Improve overall security posture

Frameworks cover both:

- Physical Security
- Virtual (Cyber) Security

---

## Why Security Frameworks Matter

Security frameworks help organizations:

- Prevent cyber attacks
- Detect security incidents
- Respond to security breaches
- Recover after attacks

Common threats they help defend against include:

- Phishing
- Social engineering
- Ransomware
- Data breaches

---

## Employee Awareness

A major purpose of security frameworks is educating employees to:

- Recognize phishing emails
- Identify suspicious activity
- Report incidents quickly
- Protect organizational assets
- Protect customer information

Human awareness is one of the strongest security defenses.

---

## Role of a Security Analyst

Security analysts use security frameworks to:

- Implement security policies
- Follow organizational procedures
- Educate employees
- Monitor security risks
- Respond to incidents
- Help reduce the likelihood of successful attacks

---

# Security Controls

## What are Security Controls?

Security controls are safeguards designed to reduce specific security risks.

Without proper controls, organizations may suffer:

- Financial losses
- Data breaches
- Reputation damage
- Unauthorized access
- Fraud

---

# Types of Security Controls

## 1. Encryption

### Definition

Encryption converts readable information (**plaintext**) into an unreadable format (**ciphertext**).

### Purpose

- Protect confidential information
- Prevent unauthorized users from reading data
- Ensure data confidentiality

---

## 2. Authentication

### Definition

Authentication verifies the identity of a user or system.

### Common Authentication Methods

- Username and password
- Multi-Factor Authentication (MFA)
- Fingerprint
- Face recognition
- Security keys

### Goal

Confirm that the user is who they claim to be.

---

## 3. Authorization

### Definition

Authorization determines what an authenticated user is allowed to access.

### Examples

- Employee can view payroll but cannot edit it.
- Student can access course materials but not instructor records.
- Administrator has full system permissions.

### Goal

Ensure users access only the resources they are permitted to use.

---

# Security Workflow

```text
User Login
      │
      ▼
Authentication
(Who are you?)
      │
      ▼
Authorization
(What can you access?)
      │
      ▼
Access Granted
```

# Key Differences

| Concept | Purpose |
|----------|---------|
| Security Framework | Provides guidelines and best practices for protecting an organization |
| Security Control | A specific safeguard that reduces security risks |
| Encryption | Protects data confidentiality |
| Authentication | Verifies identity |
| Authorization | Grants access permissions |

# Real-World Example

A company uses:

- **Security Framework:** NIST Cybersecurity Framework to establish security policies.
- **Encryption:** Protect customer credit card information.
- **Authentication:** Employees log in using Multi-Factor Authentication (MFA).
- **Authorization:** Only HR personnel can access employee salary records.

Together, these frameworks and controls significantly improve the organization's security posture.
