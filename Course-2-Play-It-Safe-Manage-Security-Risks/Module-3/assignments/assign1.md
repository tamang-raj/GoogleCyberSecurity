## Question 1

### Question
**Which log source records events related to websites, emails, and file shares, as well as password and username requests?**

- Network
- ✅ **Server**
- Firewall
- Receiving

### ✔ Correct Answer
**Server**

### Explanation
Server logs record events related to:

- Website activity
- Email services
- File-sharing services
- Login requests
- Username and password requests
- User authentication events

These logs are essential for monitoring authentication and investigating security incidents.

## Question 2

### Question
**Fill in the blank: A security information and _____ management (SIEM) tool is an application that collects and analyzes log data to monitor critical activities in an organization.**

- emergency
- employee
- efficiency
- ✅ **event**

### ✔ Correct Answer
**Event**

### Explanation
**SIEM** stands for **Security Information and Event Management**.

A SIEM platform:

- Collects logs from multiple sources
- Correlates security events
- Detects suspicious activities
- Generates alerts
- Supports incident investigations

Examples include:

- Splunk Enterprise
- Splunk Cloud
- Google Chronicle

## Question 3

### Question
**A security professional evaluates a software application by reviewing key technical attributes including response time, availability, and failure rate. What are they using to assess performance?**

- Index standards
- ✅ **Metrics**
- Models
- Cloud tools

### ✔ Correct Answer
**Metrics**

### Explanation
**Metrics** are measurable values used to evaluate system or application performance.

Common SIEM metrics include:

- Response time
- Availability
- Failure rate
- Number of security alerts
- Failed login attempts
- Incident response time

SIEM dashboards use these metrics to help analysts monitor system health and security.

## Question 4

### Question
**Fill in the blank: SIEM tools must be configured and _____ to meet each organization's unique security needs.**

- ✅ **customized**
- centralized
- reviewed
- indexed

### ✔ Correct Answer
**Customized**

### Explanation
Every organization has different:

- Security policies
- Infrastructure
- Assets
- Compliance requirements
- Threat landscape

Therefore, SIEM tools must be **configured and customized** to:

- Collect relevant logs
- Create custom detection rules
- Display useful dashboard metrics
- Generate organization-specific alerts
