# Question 1

## Question
**Playbooks are permanent, best-practice documents, so a security team should not make changes to them.**

### Options
- True
- **✅ False**

### Explanation
Playbooks are **living documents** that are regularly updated to address new cyber threats, vulnerabilities, organizational changes, and lessons learned from previous incidents.

# Question 2

## Question
**A business recently experienced a security breach. Security professionals are currently restoring the affected data using a clean backup that was created before the incident. What playbook phase does this scenario describe?**

### Options
- Detection and Analysis
- Post-Incident Activity
- **✅ Eradication and Recovery**
- Containment

### Explanation
The **Eradication and Recovery** phase focuses on removing all traces of the attack and restoring systems and data to a secure operational state, often using clean backups.

# Question 4

## Question
**Which action can a security analyst take when they are assessing a SIEM alert?**

### Options
- **✅ Analyze log data and related metrics**
- Isolate an infected network system
- Restore the affected data with a clean backup
- Create a final report

### Explanation
During the **assessment** stage of a SIEM alert, analysts review **log data**, security events, and related **metrics** to determine whether the alert represents a genuine security incident and to understand its severity.

# Key Concepts

## Playbooks
- Living documents that are continuously updated
- Standardize incident response
- Reduce response time
- Improve communication
- Ensure compliance

---

## Incident Response Phases

1. **Preparation**
   - Develop procedures
   - Train users
   - Prepare tools

2. **Detection and Analysis**
   - Detect suspicious activity
   - Analyze alerts
   - Determine impact

3. **Containment**
   - Stop the attack from spreading
   - Isolate affected systems

4. **Eradication and Recovery**
   - Remove malware
   - Eliminate attacker access
   - Restore systems using clean backups

5. **Post-Incident Activity**
   - Document lessons learned
   - Improve playbooks
   - Create reports

6. **Coordination**
   - Communicate with stakeholders
   - Share incident information
   - Ensure compliance

# Summary

- Playbooks are **living documents** that require regular updates.
- **Eradication and Recovery** restores systems after an incident.
- **Coordination** focuses on communication and reporting after resolution.
- The first step in assessing a **SIEM alert** is analyzing **log data and related metrics** to determine whether the alert is valid.
