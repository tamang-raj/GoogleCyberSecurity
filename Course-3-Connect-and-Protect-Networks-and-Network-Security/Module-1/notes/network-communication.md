# 🌐 Network Communication

Networks allow computers and other devices to communicate by exchanging information.

Organizations use networks to:

- Share files
- Access websites
- Send emails
- Connect devices
- Communicate across locations

While networks enable communication, they can also introduce security vulnerabilities that attackers may exploit.

---

# 📦 What is a Data Packet?

A **data packet** is the basic unit of information transmitted across a network.

Instead of sending one large message, networks divide information into smaller packets that travel independently to the destination.

Each packet contains information that helps it reach the correct device.

---

# ✉️ Structure of a Data Packet

A data packet is similar to sending a physical letter through the mail.

It consists of three main parts:

## 1. Header

The **header** contains routing information that directs the packet to its destination.

It includes:

- Source IP address
- Destination MAC address
- Protocol number

The header tells network devices where the packet came from and where it should go.

---

## 2. Body (Payload)

The **body**, also called the **payload**, contains the actual message or data being transmitted.

Examples include:

- Email content
- Files
- Webpage data
- Images
- Messages

---

## 3. Footer

The **footer** signals the end of the packet.

It lets the receiving device know that the complete packet has been received.

---

# 📊 Network Performance

Monitoring network performance helps organizations maintain efficient and secure communication.

Two important performance measurements are:

## Bandwidth

**Bandwidth** is the amount of data that can be received over a network per second.

Higher bandwidth allows more data to be transferred simultaneously.

---

## Speed

**Speed** refers to the rate at which data packets are received across the network.

Consistent network speed helps ensure reliable communication.

Unexpected changes in bandwidth or speed may indicate network issues or potential security threats.

---

# 🔍 Packet Sniffing

**Packet sniffing** is the process of capturing and inspecting data packets as they travel across a network.

Security professionals use packet sniffing to:

- Monitor network communication
- Analyze traffic
- Investigate suspicious activity
- Detect potential security issues

Packet sniffing helps identify abnormal network behavior that may indicate an attack.

---

# 🔐 Why It Matters for Cybersecurity

Understanding how data moves across a network helps security analysts:

- Monitor network activity
- Detect unusual traffic patterns
- Identify potential attacks
- Investigate security incidents
- Improve overall network security

Changes in network performance can serve as indicators of malicious activity.

# 📚 Key Terms

| Term | Definition |
|------|------------|
| Network | A collection of connected devices that communicate with one another |
| Data Packet | The basic unit of information transmitted across a network |
| Header | Contains addressing and protocol information used for routing |
| Payload (Body) | The actual data or message being transmitted |
| Footer | Indicates the end of a data packet |
| Bandwidth | The amount of data that can be received over a network per second |
| Speed | The rate at which data packets are received across a network |
| Packet Sniffing | The practice of capturing and inspecting network data packets |

# 📝 Key Takeaways

- Networks communicate by sending **data packets**.
- A data packet contains a **header**, **body (payload)**, and **footer**.
- The **header** includes addressing and protocol information.
- The **payload** carries the actual message or data.
- The **footer** signals the end of the packet.
- **Bandwidth** measures how much data can be transferred over a network.
- **Speed** measures how quickly packets are received.
- **Packet sniffing** allows security professionals to monitor and analyze network traffic for potential threats.
